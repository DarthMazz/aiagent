# スキル

このファイルはAIエージェントの専門的なスキルと能力を定義します。

## Python開発スキル

### コード解析スキル

- **コードレビュー**: バグ、セキュリティ問題、論理エラーを分析
  - PEP 8準拠チェック
  - 型ヒントの適切性確認
  - エラーハンドリングの検証

- **パターン認識**: コードの一般的なパターンとアンチパターンを識別
  - Pythonic なコードスタイルの推奨
  - デザインパターンの適用提案

- **依存関係分析**: ファイル間の依存関係を理解・追跡
  - モジュール間の結合度チェック
  - 循環インポートの検出

### 開発スキル

- **テスト生成**: pytestを使用した包括的なテストケース作成
  - ユニットテスト作成
  - フィクスチャの設計
  - パラメータ化テストの実装
  - モックとパッチの活用

- **リファクタリング**: 機能を維持しながらコード構造を改善
  - 関数・クラスの分割
  - 重複コードの削減
  - 型ヒントの追加
  - Docstringの充実

- **ドキュメンテーション**: コードドキュメントの生成と保守
  - Docstring（Google/NumPy形式）の記述
  - README.md の作成・更新
  - 型ヒントによる自己文書化

### コード品質スキル

- **Lintチェック**: ruffを使用した静的解析
  - `uv run ruff check .` の実行
  - Lintエラーの修正提案
  - 自動修正可能な問題の特定

- **フォーマット**: ruffを使用したコード整形
  - `uv run ruff format .` の実行
  - 一貫したコードスタイルの維持
  - Black互換フォーマットの適用

- **型チェック**: 型ヒントの検証（必要に応じてmypyを使用）

### 依存関係管理スキル

- **パッケージ管理**: uvを使用した依存関係の管理
  - `uv add` でパッケージ追加
  - `uv sync` で依存関係同期
  - pyproject.toml と uv.lock の維持

- **仮想環境管理**: プロジェクトごとの環境分離
  - `uv venv` で環境作成
  - 環境の適切なアクティベート

## ドメイン駆動設計（DDD）スキル

### 戦略的設計スキル

- **ユビキタス言語の抽出**: ドメインエキスパートとの対話から用語を抽出
  - ビジネス用語の特定と文書化
  - コードとドキュメントへの一貫した適用
  - 用語集の作成と維持

- **境界づけられたコンテキストの識別**: ドメインを適切なコンテキストに分割
  - ビジネス機能ごとの境界設定
  - コンテキスト間の関係性の分析
  - サブドメイン（コア、サポート、汎用）の分類

- **コンテキストマップの作成**: コンテキスト間の関係を可視化
  - 統合パターンの選択と適用
  - 上流/下流関係の明確化
  - チーム間の協力パターンの定義

- **ドメインモデルの洗練**: 継続的なモデルの改善
  - イベントストーミングの実施
  - モデルの深い洞察の獲得
  - リファクタリングによる表現力の向上

### 戦術的設計スキル

- **エンティティの設計**: アイデンティティを持つオブジェクトの実装
  - 一意性を保証するID設計
  - ライフサイクル管理
  - 等価性メソッドの実装
  ```python
  from dataclasses import dataclass
  from uuid import UUID
  
  @dataclass
  class Order:
      id: UUID
      customer_id: UUID
      # ビジネスロジックをメソッドとして実装
  ```

- **値オブジェクトの設計**: 不変で属性で識別されるオブジェクト
  - 不変性の保証（`frozen=True`）
  - バリデーションロジックの組み込み
  - 副作用のないメソッド実装
  ```python
  from dataclasses import dataclass
  
  @dataclass(frozen=True)
  class Email:
      address: str
      
      def __post_init__(self):
          if '@' not in self.address:
              raise ValueError("Invalid email")
  ```

- **集約の設計**: 一貫性境界を持つオブジェクトの集合
  - 集約ルートの特定
  - トランザクション境界の定義
  - 不変条件の保護
  - 集約サイズの最適化

- **リポジトリの設計**: 永続化の抽象化
  - インターフェース（Protocol/ABC）の定義
  - コレクション指向のメソッド設計
  - 仕様パターンの適用
  ```python
  from abc import ABC, abstractmethod
  from typing import Protocol
  
  class OrderRepository(Protocol):
      def find_by_id(self, order_id: UUID) -> Order | None: ...
      def save(self, order: Order) -> None: ...
  ```

- **ドメインサービスの設計**: エンティティに属さないロジック
  - ステートレスなサービスの実装
  - 複数集約にまたがる操作
  - ドメイン知識の表現
  ```python
  class PricingService:
      def calculate_total(self, order: Order, discount: Discount) -> Money:
          # 価格計算ロジック
          ...
  ```

- **ドメインイベントの設計**: ドメインでの重要な出来事
  - イベントの識別と命名（過去形）
  - イベントペイロードの設計
  - イベントハンドラーの実装
  ```python
  @dataclass(frozen=True)
  class OrderPlaced:
      order_id: UUID
      customer_id: UUID
      placed_at: datetime
  ```

- **ファクトリの設計**: 複雑なオブジェクト生成
  - 集約の生成ロジックのカプセル化
  - ビジネスルールに基づいた生成
  - 不変条件の保証

### アーキテクチャスキル

- **ヘキサゴナルアーキテクチャの構築**: レイヤー分離の実装
  - ドメイン層の独立性確保
  - アプリケーション層でのユースケース実装
  - インフラ層での技術的詳細の実装

- **ポート（インターフェース）の定義**: 抽象化の設計
  - Protocol または ABC を使用
  - 入力ポート（アプリケーションサービス）
  - 出力ポート（リポジトリ、外部サービス）

- **アダプターの実装**: 具体的な技術実装
  - データベースアダプター（SQLAlchemy、MongoDBなど）
  - APIアダプター（FastAPI、Flaskなど）
  - メッセージングアダプター（RabbitMQ、Kafkaなど）

- **依存性逆転の原則（DIP）の適用**: 依存関係の制御
  - インターフェースへの依存
  - 依存性注入の実装
  - 技術的詳細の隔離

### モデリングスキル

- **ドメインエキスパートとの対話**: ビジネス知識の獲得
  - 適切な質問による知識の引き出し
  - ユビキタス言語の共創
  - ビジネスルールの明確化

- **イベントストーミングの実施**: 協働モデリング手法
  - ドメインイベントの洗い出し
  - コマンドとアクターの特定
  - 集約と境界の発見

- **モデルの継続的な改善**: リファクタリングと深化
  - より深い洞察の反映
  - 暗黙的な概念の明示化
  - コードとモデルの同期

- **ドメイン知識の文書化**: 知識の共有と保存
  - ユビキタス言語の用語集
  - コンテキストマップ
  - アーキテクチャ決定記録（ADR）

## プロジェクト管理スキル

- **タスク計画**: 複雑なタスクを実行可能なステップに分解
  - Python開発のベストプラクティスに従った計画
  - テスト駆動開発（TDD）の適用

- **進捗追跡**: 実装の進捗状況を監視・報告
  - テストカバレッジの追跡
  - Lintエラー数の監視

- **リスク評価**: 問題発生前に潜在的な課題を特定
  - 依存関係の競合リスク
  - セキュリティ脆弱性の検出

## コミュニケーションスキル

- **簡潔な報告**: 明確で簡潔なステータス更新を提供
- **技術的説明**: 複雑な概念を理解しやすい言葉で説明
- **対話的ガイダンス**: 複数ステップのプロセスを通じてユーザーを案内
